# Contributing to FormBharat

Thank you for your interest in contributing to FormBharat! This document provides guidelines for contributing to the project.

## Development Setup

1. Fork the repository
2. Clone your fork: `git clone https://github.com/your-username/form-bharat.git`
3. Install dependencies: `npm install`
4. Set up environment variables (see SETUP.md)
5. Run migrations: `npx prisma migrate dev`
6. Start dev server: `npm run dev`

## Code Style

- Use TypeScript for all new code
- Follow existing code formatting
- Use meaningful variable and function names
- Add comments for complex logic
- Keep components small and focused

## Commit Messages

- Use clear, descriptive commit messages
- Start with a verb: "Add", "Fix", "Update", "Remove"
- Examples:
  - "Add payment field type"
  - "Fix form submission validation"
  - "Update dashboard UI layout"

## Pull Request Process

1. Create a new branch: `git checkout -b feature/your-feature-name`
2. Make your changes
3. Test thoroughly
4. Commit with clear messages
5. Push to your fork
6. Create a Pull Request with description of changes

## Feature Requests

- Open an issue describing the feature
- Explain the use case
- Discuss implementation approach

## Bug Reports

Include:
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots if applicable
- Environment (browser, OS, etc.)

## Code of Conduct

- Be respectful and inclusive
- Welcome newcomers
- Focus on constructive feedback
- Help others learn and grow

## Questions?

Open an issue or start a discussion on GitHub.
